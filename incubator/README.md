# Incubator

Active questions live here. An entry may be one note, a runnable probe, or a
substantial system. It does not need to graduate or follow a lifecycle.

When starting an idea, create `incubator/<idea-name>/`. If useful, ask:

- What am I wondering?
- What could I try?
- What did I notice?

## A useful reading order

This order was seeded from each subject's earliest Git history, then adjusted
for comprehension. It is intentionally curated here instead of encoded in
directory names.

1. [`hello-world`](./hello-world/) — the smallest guarded effect.
2. [`dependency-upgrade`](./dependency-upgrade/) — bounded workflow control.
   [`dependency-upgrade-instrumented`](./dependency-upgrade-instrumented/) adds
   observability to the same subject.
3. [`email-review-send`](./email-review-send/) — authority around a sensitive effect.
4. [`crust`](./crust/) — durable, Pi-native state and evidence around Pocock.
5. [`podcast-producer`](./podcast-producer/) — a broader workflow case study.
