# Roadmap

This is a thinking horizon, not a release plan or issue backlog.

## Now

- Consolidate active harness research here.
- Make Pi Agent Core the explicit substrate.
- Use the current Crust work as an active, demanding case study without treating
  Pocock as the canonical workflow.
- Keep orientation simple enough that people and agents can find the relevant
  idea without copying the whole repository.

## Next

- Probe the [smallest credible harnessed workflow](https://github.com/bioshazard/harness-engineering/issues/1).
- Probe narrow [draft-only email authority](https://github.com/bioshazard/harness-engineering/issues/13).
- Let concrete Pi-native work reveal which state, evidence, verification, and
  composition patterns recur.

## Later

- Distill explanations that help across multiple incubator entries into `docs/`.
- Promote implementation into `src/` when duplicated use creates real friction.
- Improve the teaching path after the examples reveal a useful order.

## Not optimizing for

- A generic or portable harness framework.
- An npm-installable library or supported CLI product.
- A universal Pocock workflow.
- A complete taxonomy, standard model, actor-model doctrine, or formal lifecycle.
- Dates, releases, or broad compatibility.
